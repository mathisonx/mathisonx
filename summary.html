建设历程


前言

      本站构建环境基于Mac OS Mojave。其他平台的操作构建方法类似。为简化文档以及使得流程更加清晰，部分内容我会引用社区文档。

一、GitHub账户

      由于服务器的属地因素，静态网站的访问速度可能不甚理想。
      GitHub账户注册参考CSDN社区文档： https://blog.csdn.net/qq_41782425/article/details/85179912

二、域名申请

      注意：在GitHub中，当您开启GitHub Pages功能之后，会生成个人主页网址，您可以凭借此网址访问您的个人主页，因此域名并不是必须的，但是如果您想通过个人域名访问个人主页，可以使用域名解析的方式。
      域名申请参考腾讯云社区文档。 https://cloud.tencent.com/developer/article/1497409

三、Git环境配置

1、Git安装：https://sourceforge.net/projects/git-osx-installer/
2、基于Mac OS，当您完成Git的安装后，可以在终端中查看是否安装成功，方法如下：
      使用键盘Command+space调出聚焦搜索，输入Terminal.app打开终端，输入git version，如果出现git的版本代码，如 git version 2.15.0，说明已经成功安装git。
3、Git的配置：目的在于建立本机与GitHub服务器的通信，参考社区文档：https://blog.csdn.net/qq_37721114/article/details/105809948

四、本地与远程仓库

1、首先在GitHub中建立仓库，建立方法参考（一）中的社区文档。
2、在本地建立一个文件夹，可以选择在桌面建立，文件夹创建完成后，终端路径到本文件夹，输入 git init，初始化仓库，本地仓库建立完成。


五、本地代码的发布

1、准备好所有的代码文件，存放至本地仓库，使用终端命令
git add --all //添加所有文件
git commit -m “发布” //提交文件到本地
git push origin master //提交文件到远程仓库
